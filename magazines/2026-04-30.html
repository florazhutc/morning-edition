To create a fundamentally better editor, we had to invent a new approach to building desktop software. Our previous editor, Atom , was built as a fork of Chromium, spawning the Electron framework in the process. Electron eventually became the foundation of VS Code (which today seems to be forked into a new AI code editor every other week). Web technology offered an easy path to shipping flexible software, but it also imposed a ceiling. No matter how hard we worked, we couldn't make Atom better than the platform it was built on.

Actionable Insight

Actionable Insight

Software Safety 🛡️

Rust's Blind Spots: 44 Coreutils Vulnerabilities Evaded Borrow Checker and Clippy

Bugs Rust Won't Catch | corrode Rust Consulting In April 2026, Canonical disclosed 44 CVEs in uutils, the Rust reimplementation of GNU coreutils that ships by default since 25.10. Most of them came out of an external audit commissioned ahead of the 26.04 LTS. I read through the list and thought there’s a lot to learn from it. What’s notable is that all of these bugs landed in a production Rust codebase, written by people who knew what they were doing, and none of them were caught by the borrow checker, clippy lints , or cargo audit .

Actionable Insight

This incident underscores that even highly memory-safe languages like Rust, with their robust static analysis tools, are not a panacea for all software vulnerabilities. The discovery of numerous CVEs unflagged by Rust's core safety features highlights the persistent need for comprehensive external audits and rigorous testing to uncover logic errors, API misuse, and subtle semantic bugs that bypass type systems and borrow checkers. It's a critical reminder that true software reliability extends far beyond memory safety guarantees.

Community Voice

The community largely agrees that the "bugs Rust won't catch" stem not from Rust's language features, but from developers lacking deep experience with intricate Unix APIs and their security implications, leading to issues like TOCTOU races. There is significant criticism towards the Rust coreutils rewrite for failing to incorporate decades of hard-won knowledge from the original C implementations, and also towards Ubuntu for allowing such "amateur" attempts into production.
